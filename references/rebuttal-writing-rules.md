# Rebuttal Writing Rules

## Core Orientation
- Stay courteous, objective, and specific.
- Acknowledge the comment, state the stance, then move quickly to substance.
- Default to response-letter text that can be pasted directly into the rebuttal.
- If the user provides rough Chinese notes, convert them into direct, polite, publication-ready English instead of translating literally.

## Standard Response Skeleton
1. Open with one sentence of thanks.
2. State the stance: agree, partially agree, or respectfully clarify.
3. State the revision or clarification directly.
4. Provide reasoning in ordered points if the comment contains multiple issues.
5. Point to the manuscript location when the location is known.
6. Close with a concise hope statement only when useful.

## Tone Rules
- Use calm, non-defensive language.
- Concede valid points plainly.
- When disagreeing or partially disagreeing, keep the disagreement narrow and evidence-based.
- Avoid emotional language, vague self-praise, or exaggerated apology.
- Use a more ceremonial level of politeness in editor- or AE-level overall responses than in point-by-point replies.

## Distilled Behavioral Norms
- Thank briefly, then move to substance.
- Use `Specifically` to introduce concrete revisions.
- Use `First`, `Second`, and `Third` only when directly matching the structure of the reviewer comment.
- Separate `what we changed` from `why the change addresses the concern`.
- If a suggestion cannot be fully implemented, explain the constraint directly and still provide the strongest feasible revision.
- When novelty is questioned, clarify it from the problem, methodological, and practical perspectives instead of repeating generic novelty claims.
- When assumptions are questioned, explain why they are standard, what scope they define, and where the limitation is acknowledged.
- When clarity is questioned, state exactly what text, figure, appendix, or experiment was added.
- When known, include a revision trace: what changed, why it addresses the concern, and where the change appears in the revised manuscript.
- If the strongest response is to narrow scope, delete a weak claim, or remove an unsupported proposition, say so directly and explain the consequence for the remaining analysis.
- When a request is outside the scope of the paper or unreasonable for the current revision, acknowledge the value of the suggestion, define the scope boundary clearly, and provide the strongest evidence-based response that remains within scope.

## Common Response Patterns

### Clarification
- `Thank you for this insightful comment. We have clarified ... Specifically, ...`

### Added Experiment Or Analysis
- `Thank you for this helpful suggestion. We have followed this suggestion and added ... Specifically, ...`

Common additions include benchmark comparisons, ablation studies, sensitivity analysis, larger-scale instances, external datasets, complexity analysis, and limitation discussion when these directly answer the concern.

### Partial Agreement
- `We agree that ... However, ... To address this concern, we have ...`

### Respectful Disagreement Or Scope Limitation
- `We appreciate this suggestion. However, we respectfully note that [requested item] is beyond the scope of the current study because [reason]. To address the concern as far as possible within scope, we have [clarification / additional discussion / focused experiment].`

### Novelty Clarification
- `Below we clarify the contribution from the problem perspective and the methodological perspective.`

### Multiple Sub-Issues
- `This comment contains three issues, and we address them in turn below.`

### Limitation And Future Work
- `We agree that this is a limitation of the current study. We now discuss this point in the manuscript and identify it as an important direction for future work.`

### Reject And Resubmit Context
- In reject-and-resubmit or major-revision contexts, explicitly distinguish between `what has been newly added`, `what has been clarified`, and `what remains outside the scope of the revised manuscript`.

### Confusion Repair
- `We are sorry for the confusion.`
- `We apologize for any confusion caused.`
- `You are correct that ...`

## Manuscript-Location Rules
- Use `Please refer to ...` only when the target section, page, line, figure, table, or appendix is known from the actual manuscript context.
- Never invent page, line, or section references.
- If the exact location is unknown, refer to the revised section by name or say `in the revised manuscript` without false precision.
- When the location is known, prefer concrete traceability such as section, page, line, appendix, figure, table, screenshot, highlighting convention, or code link if those artifacts are actually available.

## Do Not
- Do not sound combative.
- Do not over-thank in every sentence.
- Do not claim a change was made if it was not.
- Do not substitute future work for a solvable reviewer concern.
- Do not hide disagreement; state it politely and precisely.
- Do not accept an unreasonable request automatically if it would distort the paper's scope or weaken the argument.

## Reusable Patterns
- Polite opening:
  - `Thank you for this insightful comment.`
- Stronger opening for editor or AE:
  - `Thank you very much for your encouragement and for the opportunity to revise the manuscript.`
- State action:
  - `We have carefully revised the manuscript accordingly.`
- Clarify mechanism:
  - `Specifically, we now explain that ...`
- Explain structure:
  - `There are two issues in this comment, and we address them separately below.`
- Domain-fit clarification:
  - `We would like to respectfully clarify that the primary contribution of our work lies in the transportation domain.`
- Scope boundary:
  - `While this suggestion is valuable, we respectfully note that it goes beyond the scope of the current study.`
- Point to manuscript:
  - `Please refer to [Section / Figure / Table / Appendix / Lines] for details.`

## Mini Example
Reviewer comment:
- `The authors should evaluate the method on a completely different network type and redesign the optimization model accordingly.`

Response pattern:
- `Thank you for this valuable suggestion. We agree that evaluation on additional network types would be useful. However, we respectfully note that redesigning the optimization model for a different network type would substantially expand the scope of the current study. To address this concern within scope, we have clarified the applicability boundaries of our model and added a limitation discussion in the revised manuscript. Please refer to [Section / Appendix / Lines] for details.`

## Paste-Ready Default
- Output finished rebuttal text directly.
- Use placeholders such as `[Section X]` or `[Lines X-Y]` only when the user explicitly asks for a template rather than a finished response.
